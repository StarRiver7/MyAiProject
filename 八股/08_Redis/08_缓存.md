# 缓存（核心面试知识）

---

## 一、缓存概述

> 缓存是一种提高系统性能的技术，将热点数据存储在高速存储介质中

### 1.1 缓存的作用

```
1. 提高访问速度：减少数据库访问
2. 降低系统负载：减轻数据库压力
3. 提高系统可用性：缓存服务比数据库更稳定
4. 降低网络延迟：缓存通常部署在靠近应用的地方
```

### 1.2 Redis作为缓存的优势

```
- 内存存储，速度快
- 支持多种数据结构
- 支持持久化
- 支持主从复制和集群
- 丰富的客户端库
```

## 二、缓存策略

### 2.1 缓存读写策略

#### 1. Cache-Aside（旁路缓存）

```
读：
1. 先查缓存
2. 缓存命中，返回
3. 缓存未命中，查数据库
4. 将数据写入缓存
5. 返回数据

写：
1. 更新数据库
2. 失效缓存

优点：简单直接
缺点：可能出现缓存不一致
```

#### 2. Read-Through（读透缓存）

```
读：
1. 先查缓存
2. 缓存命中，返回
3. 缓存未命中，缓存服务查数据库
4. 缓存服务更新缓存
5. 返回数据

优点：应用无需关心缓存逻辑
缺点：实现复杂
```

#### 3. Write-Through（写透缓存）

```
写：
1. 应用更新缓存
2. 缓存服务更新数据库

优点：数据一致性好
缺点：写操作延迟高
```

#### 4. Write-Back（写回缓存）

```
写：
1. 应用更新缓存
2. 缓存服务异步更新数据库

优点：写操作性能高
缺点：可能丢失数据
```

## 三、缓存问题

### 3.1 缓存穿透

```
定义：请求不存在的数据，导致缓存和数据库都被查询

原因：
- 恶意攻击
- 业务逻辑错误
- 数据被删除

解决方案：
1. 布隆过滤器
2. 缓存空值
3. 限流
4. 校验参数
```

### 3.2 缓存击穿

```
定义：热点数据过期，导致大量请求同时查询数据库

原因：
- 热点数据缓存过期
- 缓存服务重启

解决方案：
1. 设置热点数据永不过期
2. 后台刷新缓存
3. 互斥锁
4. 双层缓存
```

### 3.3 缓存雪崩

```
定义：大量缓存同时过期，导致数据库压力骤增

原因：
- 缓存集中过期
- 缓存服务宕机

解决方案：
1. 分散过期时间
2. 多级缓存
3. 服务降级
4. 熔断
5. 限流
```

### 3.4 缓存一致性

```
定义：缓存数据与数据库数据不一致

原因：
- 先更新数据库，后失效缓存（存在时间窗口）
- 先失效缓存，后更新数据库（存在时间窗口）
- 并发更新

解决方案：
1. 延迟双删
2. 消息队列
3. 最终一致性
4. 读写锁
```

## 四、缓存设计

### 4.1 键设计

```
1. 命名规范：业务:模块:id
   - 例如：user:profile:10001

2. 键长度：
   - 控制在100字节以内
   - 避免过长的键名

3. 键类型：
   - 使用合适的数据结构
   - 例如：用户信息用Hash，列表用List
```

### 4.2 过期时间

```
1. 根据业务设置合理的过期时间
2. 避免集中过期
3. 考虑热点数据

示例：
- 会话数据：30分钟
- 商品信息：1小时
- 热点数据：永不过期（后台刷新）
```

### 4.3 内存管理

```
1. 设置合理的maxmemory
2. 选择合适的内存淘汰策略
3. 定期清理过期数据
4. 监控内存使用

推荐策略：
- 热点数据：allkeys-lru
- 有过期时间的数据：volatile-lru
```
