## 一、Redis集群概述

> Redis Cluster是Redis的分布式解决方案，用于实现数据的自动分片和高可用

### 1.1 特点

```
1. 数据自动分片：将数据分布到多个节点
2. 高可用：节点故障自动转移
3. 水平扩展：支持添加节点
4. 无中心架构：所有节点平等
5. 客户端分片：客户端直接连接所有节点
```

---

## 二、集群原理

### 2.1 数据分片

```
Redis Cluster使用哈希槽（Hash Slot）进行数据分片：
- 总共有16384个哈希槽（0-16383）
- 每个键通过CRC16算法计算哈希值
- 哈希值对16384取模，得到槽位
- 每个节点负责一部分槽位
```

### 2.2 节点角色

| 角色 | 说明 |
|------|------|
| **主节点** | 负责处理槽位，存储数据 |
| **从节点** | 主节点的备份，主节点故障时升级 |

### 2.3 集群拓扑

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  主节点1   │     │  主节点2   │     │  主节点3   │
│  槽0-5460  │     │  槽5461-10922 │  │  槽10923-16383 │
└─────┬───────┘     └─────┬───────┘     └─────┬───────┘
      │                   │                   │
┌─────▼───────┐     ┌─────▼───────┐     ┌─────▼───────┐
│  从节点1   │     │  从节点2   │     │  从节点3   │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 三、集群配置

### 3.1 配置文件

```bash
# 集群节点配置

# 开启集群模式
cluster-enabled yes

# 集群配置文件
cluster-config-file nodes-6379.conf

# 集群节点超时时间
cluster-node-timeout 15000

# 集群从节点投票超时时间
cluster-slave-validity-factor 10

# 从节点迁移
cluster-migration-barrier 1

# 集群复制超时
cluster-replica-timeout 60
```

### 3.2 创建集群

```bash
# 准备节点（6个节点，3主3从）
# 端口：6379, 6380, 6381, 6382, 6383, 6384

# 使用redis-cli创建集群
redis-cli --cluster create \
127.0.0.1:6379 \
127.0.0.1:6380 \
127.0.0.1:6381 \
127.0.0.1:6382 \
127.0.0.1:6383 \
127.0.0.1:6384 \
--cluster-replicas 1

# 参数说明：
# --cluster-replicas 1：每个主节点有1个从节点
```

### 3.3 启动集群

```bash
# 启动每个节点
redis-server redis-6379.conf
redis-server redis-6380.conf
redis-server redis-6381.conf
redis-server redis-6382.conf
redis-server redis-6383.conf
redis-server redis-6384.conf

# 查看集群状态
redis-cli -c -p 6379 cluster info
```

## 四、集群操作

### 4.1 查看集群信息

```bash
# 查看集群信息
redis-cli -c -p 6379 cluster info

# 查看节点信息
redis-cli -c -p 6379 cluster nodes

# 查看槽位分配
redis-cli -c -p 6379 cluster slots
```

### 4.2 添加节点

```bash
# 添加主节点
redis-cli --cluster add-node new_node_ip:port existing_node_ip:port

# 添加从节点
redis-cli --cluster add-node new_node_ip:port existing_node_ip:port --cluster-slave

# 添加指定主节点的从节点
redis-cli --cluster add-node new_node_ip:port existing_node_ip:port --cluster-slave --cluster-master-id master_id
```

### 4.3 槽位重分配

```bash
# 重新分配槽位
redis-cli --cluster reshard existing_node_ip:port

# 按照比例分配
redis-cli --cluster reshard existing_node_ip:port --cluster-from all --cluster-to target_node_id --cluster-slots 100 --cluster-yes
```

### 4.4 删除节点

```bash
# 删除从节点
redis-cli --cluster del-node existing_node_ip:port slave_node_id

# 删除主节点（需要先迁移槽位）
# 1. 先迁移槽位
redis-cli --cluster reshard existing_node_ip:port --cluster-from master_node_id --cluster-to other_master_id --cluster-slots 16384 --cluster-yes

# 2. 删除节点
redis-cli --cluster del-node existing_node_ip:port master_node_id
```

## 五、故障转移

### 5.1 主节点故障

```
1. 从节点检测到主节点超时
2. 从节点向其他主节点发送故障检测请求
3. 超过半数主节点确认主节点故障
4. 从节点发起选举
5. 选举成功的从节点升级为主节点
6. 新主节点接管槽位
7. 集群恢复正常
```

### 5.2 故障转移时间

```
- 检测时间：cluster-node-timeout（默认15秒）
- 选举时间：约1-2秒
- 总共时间：约16-17秒
```

## 六、集群客户端

### 6.1 客户端连接

```python
# Python示例（使用redis-py-cluster）
from rediscluster import RedisCluster

# 启动时连接的节点
startup_nodes = [
    {'host': '127.0.0.1', 'port': '6379'},
    {'host': '127.0.0.1', 'port': '6380'},
    {'host': '127.0.0.1', 'port': '6381'}
]

# 连接集群
rc = RedisCluster(startup_nodes=startup_nodes, decode_responses=True)

# 使用
rc.set('key', 'value')
print(rc.get('key'))
```

### 6.2 客户端特点

```
- 自动发现集群拓扑
- 自动重定向到正确的节点
- 处理故障转移
- 支持读写分离
```

## 七、集群限制

### 7.1 不支持的命令

```
- 多键操作（如MSET多个键不在同一槽位）
- 事务操作（MULTI/EXEC）跨槽位
- LUA脚本跨槽位
- PUBLISH/SUBSCRIBE（只能在一个节点上）
```

### 7.2 解决方法

```
- 使用哈希标签（Hash Tag）确保多键在同一槽位
- 例如：{user:1000}:name, {user:1000}:age
- 这些键会被分配到同一个槽位
```

## 八、集群最佳实践

### 8.1 部署建议

```
1. 至少3个主节点，每个主节点至少1个从节点
2. 节点分布在不同机器上
3. 合理配置内存
4. 监控集群状态
```

### 8.2 配置建议

```bash
# 集群配置
cluster-enabled yes
cluster-config-file nodes.conf
cluster-node-timeout 15000
cluster-slave-validity-factor 10
cluster-migration-barrier 1

# 内存配置
maxmemory 4GB
maxmemory-policy allkeys-lru

# 持久化配置
appendonly yes
appendfsync everysec
```

### 8.3 监控建议

```
- 监控集群状态（cluster info）
- 监控槽位分配
- 监控节点健康状态
- 监控内存使用
- 监控网络延迟
```
