# 存储引擎（核心面试知识）

---

## 一、MySQL存储引擎概述

MySQL支持多种存储引擎，常用的有：**InnoDB** 和 **MyISAM**

### 查看存储引擎

```sql
SHOW ENGINES;                    -- 查看所有存储引擎
SHOW VARIABLES LIKE 'default_storage_engine';  -- 查看默认存储引擎
```

---

## 二、InnoDB与MyISAM对比（高频面试点）

| 特性 | InnoDB | MyISAM |
|------|--------|--------|
| **事务** | 支持事务 | 不支持事务 |
| **外键** | 支持外键 | 不支持外键 |
| **锁粒度** | 行级锁 | 表级锁 |
| **并发性能** | 高 | 低 |
| **存储空间** | 较大 | 较小 |
| **崩溃恢复** | 自动恢复 | 较慢 |
| **全文索引** | 5.6+支持 | 支持 |
| **适用场景** | 核心业务 | 查询为主的场景 |

---

## 三、InnoDB核心特性（必须掌握）

### 3.1 事务支持

- 支持ACID特性
- 自动崩溃恢复
- 四种隔离级别

### 3.2 行级锁

- 基于索引实现
- 支持高并发
- 锁开销大

### 3.3 外键约束

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);
```

### 3.4 MVCC（多版本并发控制）

- 乐观锁实现
- 读写不阻塞
- 通过undo log实现

---

## 四、MyISAM核心特性

### 4.1 表级锁

- 全表加锁
- 适合以读为主场景
- 锁开销小

### 4.2 全文索引

```sql
CREATE TABLE article (
    id INT PRIMARY KEY,
    content TEXT,
    FULLTEXT(content)
);
```

### 4.3 压缩表

- 支持表压缩
- 节省空间
- 适合归档

## 五、存储结构对比

### InnoDB

```
表空间文件：
├── ibdata1          -- 共享表空间
├── undo logs        -- Undo日志
├── temp tables      -- 临时表
└── user_id.ibd     -- 独立表空间（每表一个）

数据存储：.ibd文件（独立表空间）
```

### MyISAM

```
存储文件（每表三个）：
├── user.MYD         -- 数据文件
├── user.MYI         -- 索引文件
└── user.frm         -- 表结构文件
```

## 六、表空间

### 6.1 共享表空间 vs 独立表空间

```sql
-- 查看表空间模式
SHOW VARIABLES LIKE 'innodb_file_per_table';

-- 开启独立表空间（默认开启）
SET GLOBAL innodb_file_per_table = ON;
```

| 模式 | 优点 | 缺点 |
|------|------|------|
| 共享表空间 | 空间统一管理 | 空间回收困难 |
| 独立表空间 | 空间易回收、可压缩 | 管理分散 |

### 6.2 表空间迁移

```sql
-- 步骤：detach → 移动文件 → attach
ALTER TABLE user DETACH身后;
-- 移动 user.ibd 文件
ALTER TABLE user ATTACH身后;
```

## 七、InnoDB vs MyISAM 选择

> **选InnoDB的场景**：
> - 需要事务支持
> - 需要高并发
> - 有外键约束
> - 数据更新频繁

> **选MyISAM的场景**：
> - 以读为主
> - 很少更新
> - 全文搜索
> - 空间敏感

## 八、重要背点总结

> **面试高频考点**：
> 1. InnoDB vs MyISAM 区别（六点对比）
> 2. InnoDB特性：事务、行锁、外键、MVCC
> 3. MyISAM特性：表锁、全文索引、压缩
> 4. 表空间模式选择（共享 vs 独立）
> 5. 崩溃恢复能力对比
